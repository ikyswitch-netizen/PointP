# Torus Loops — 辺整合パズル（生成器 + solver + GUI）

m×n の正方形ピースをトーラス面（上下・左右がループ）に並べ、隣接する辺の状態が
すべて整合（σ: なし↔なし、(色c,out)↔(色c,in)）すると完成するパズルです。

## 遊び方

```
node serve.js
```

を実行して http://localhost:8123 を開く（または Claude Code の preview で `puzzle` を起動）。

- **ドラッグ**で配置、**クリック**で 90° 回転。占有セルへ落とすと入替え。
- 盤の外周マージンは反対側の写し。そこへドロップすると回り込んで置ける。
- 各辺の状態は「中心 ↔ 辺の中点」を往復する色付きの点。out は t=0 で中心から出発、
  in は半周期ズレ（t=0 で辺から中心へ）。正しく隣接すると継ぎ目をまたいで
  点の流れが連続して見え、ズレていると継ぎ目で衝突・断絶する。
- すべての継ぎ目（回り込み含む）が整合したらクリア。
  anchor は盤面に固定しないため、**盤全体の平行移動・（正方形なら）回転も正解**。
- ヘッダで盤サイズ 3×3〜6×6、色数（既定 K≈√mn）を変更可能。
  「答え」= intended 解のミニ表示、「ソルバーで解く」= 全探索 solver の解をアニメ配置。

## ファイル構成

| ファイル | 内容 |
|---|---|
| `engine.js` | 辺エンコーディング・solver（全探索の解数え上げ）・生成器。Node/ブラウザ両対応 |
| `app.js` | Canvas GUI（描画・D&D・回転・クリア判定・生成の非同期ポンプ） |
| `index.html` | ページ本体・操作 UI |
| `serve.js` | 依存なし静的サーバ（開発検証用の `POST /snap` 付き） |
| `test.js` | エンジンの検証テスト（`node test.js`） |

## 設計メモ

- **辺の状態**: `0 = なし`, `1+2c+d`（色 c、d: 0=out / 1=in）。整合は `partner(v)`
  （out↔in の対合）との一致で判定。**有向**マッチングであり無向の色合わせではない。
- **一意性の定義**: intended 解でセル (0,0) にあるピースを anchor としてセル 0・回転 0 に
  固定し、残りピースの「配置＋向き」の割当てを全探索で数えて **ちょうど 1** であること。
  - ピース実体×回転 0..3 をすべて区別する厳密カウントなので、同一ピースの swap・
    3-cycle・行/列シフト・回転対称ピースの向き替えは自動的に別解として検出される
    （伝播チェックではなく全探索。トーラスの大域平行移動・回転は anchor 固定で消える）。
  - 探索は「(西面の要求値, 北面の要求値) → 候補 (piece, rot)」のインデックス引き +
    行末/最下行での wrap 制約フィルタ。ノード上限つき（上限到達は「未確定」扱いで、
    生成器は保守側 = その削りを棄却する。採用された盤面は必ず完走した探索で count=1）。
- **生成**: ① 全 seam をランダム彩色した完全解を作り一意性を確認（だめなら作り直し）。
  ② seam をランダム順に 1 本ずつ「なし」へ落とし、一意性が保てるときだけ採用（数独式）。
  1 パスで何も削れなくなるまで反復。時間予算（6 秒）内に収束すれば **局所極小**
  （どの色付き seam も単独では消せない）で、ステータスバーに表示される。
  予算切れの場合も一意性そのものは最終全探索で検証済み（準極小、仕様の妥協点）。
- **色数の既定**: K = max(2, round(√(mn)))。情報量下限 (2K+1)² ≳ mn 由来の出発点。
