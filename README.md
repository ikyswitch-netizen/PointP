# Torus Loops — 辺整合パズル（生成器 + solver + GUI）

m×n の正方形ピースをトーラス面（上下・左右がループ）に並べ、隣接する辺の状態が
すべて整合（σ: なし↔なし、(色c,out)↔(色c,in)）すると完成するパズルです。

## 遊び方

```
node serve.js
```

を実行して http://localhost:8123 を開く（または Claude Code の preview で `puzzle` を起動）。

- **ドラッグ**で配置、**クリック**で 90° 回転。占有セルへ落とすと入替え。
- 盤の外周マージンは反対側の写し。そこへドロップすると回り込んで置ける。
- 色付きの点は「自分の中心 ↔ **隣のタイルの中心**」を往復しようとする単振動
  （振動中心 = 継ぎ目、振幅の端点 = 両タイルの中心）。ただし**自分のタイルの中でしか
  描かれない**。隣が正しければ、点が継ぎ目を越えた瞬間に相手側の点が位相の続きを
  引き継ぎ、1つの点が両中心を行き来しているように見える。
  継ぎ目で点が消えたまま帰らない・2つの点がぶつかって消える・色が変わって戻ってくるなら、
  その継ぎ目は間違い。軌道や向きを示す補助線はあえて描かない（点の動きだけが手がかり）。
- すべての継ぎ目（回り込み含む）が整合したらクリア。
  anchor は盤面に固定しないため、**盤全体の平行移動・（正方形なら）回転も正解**。

### モード

- **ステージ**（既定）: 12 ステージのカリキュラム。難易度は複数の軸
  （①足場＝固定ピース数、②置くピース数、③色数 K、④手がかりの間引き）で少しずつ上げ、
  1ステージではなるべく1軸だけ動かして崖を作らない。Stage 1〜6・9 は
  **forced move だけで（当てずっぽうなしで）解ける**ことを保証している（`forcedSolvable`）。
  - **Phase A（1〜4）** 足場ありの基礎（〜2色）: はめ込む → 向きを読む → 輪を閉じる → 静けさも手がかり
  - **Phase B（5〜6）** 色を3色に増やして置くピースを増やしつつ、足場を段階的に外す
    （固定5枚→3枚→2枚）。full hints かつ無推測なので増分は小さい
  - **Phase C（7〜9）** 足場ゼロ＝平行移動の自由 → **色を2色に落として「色が少ない難しさ」を体感**
    （同じ盤・同じ手がかりでも紛らわしくなる）→ 手がかりを間引いた演繹（無推測）
  - **Phase D（10〜12）** 4×4 → 5×5 → 6×6 の実戦。ここは先読み（推測）も要る難関
  クリアで次が解放される（進行は localStorage に保存。ソルバーでのクリアは進行に
  数えない）。手作り面の一意性（固定ピース込み）・無推測性も test.js が検証している。
- **フリー**: 盤サイズ 2×2〜6×6・色数（既定 K≈√mn）を選んでランダム生成。
  「答え」= intended 解のミニ表示、「ソルバーで解く」= 全探索 solver の解をアニメ配置。

クリア後は演出バナーが数秒で消え、金色の枠と流れ続ける点だけが残る（鑑賞モード）。

### お絵描き（思考メモ）

ツールバーの ✏️ペンで盤やピースに自由に書き込める（クリア判定には影響しない）。
- **ピースの上から書き始めた線はそのピースに付く**: ドラッグ・クリック回転・
  トレイ⇄盤の移動にすべて追従する（ローカル座標で保持、wrap の写しにも表示）。
  固定ピースにも書ける。
- 盤に書いた線は盤座標に張り付き、ウィンドウをリサイズしてもセルからずれない。
- 🧽消しゴムは線（ストローク）単位で消える。🗑で全消去。新しい問題で自動的に消える。
- ショートカット: **Shift+ドラッグ = ペン**、**Alt+ドラッグ = 消しゴム**（どのツール中でも）。
  色スウォッチを選ぶと自動でペンに切り替わる。

## 設計メモ

- **辺の状態**: `0 = なし`, `1+2c+d`（色 c、d: 0=out / 1=in）。整合は `partner(v)`
  （out↔in の対合）との一致で判定。**有向**マッチングであり無向の色合わせではない。
- **点の位相**: 周期 T の単振動で、位置 a(t)∈[0,2]（0=自中心, 1=継ぎ目, 2=隣中心）。
  out: a = 1−cos(2πt/T)、in: a = 1+cos(2πt/T)。a≤1 のときだけ描画する。
  整合した隣（out↔in）では相手の a が常に 2−a になるため、継ぎ目で位置が正確に
  接続し、1つの点の連続運動に見える。
- **一意性の定義**: intended 解でセル (0,0) にあるピースを anchor としてセル 0・回転 0 に
  固定し、残りピースの「配置＋向き」の割当てを全探索で数えて **ちょうど 1** であること。
  固定ピースのあるステージでは anchor の代わりに全固定ピース（`opts.fixed`）を
  事前配置して同じ数え上げを行う（`puzzleFromDesign`）。
  - ピース実体×回転 0..3 をすべて区別する厳密カウントなので、同一ピースの swap・
    3-cycle・行/列シフト・回転対称ピースの向き替えは自動的に別解として検出される
    （伝播チェックではなく全探索。トーラスの大域平行移動・回転は anchor 固定で消える）。
  - 探索は「(西面の要求値, 北面の要求値) → 候補 (piece, rot)」のインデックス引き +
    行末/最下行での wrap 制約フィルタ。幅 2 の盤（東西/南北の隣が同一ピースで
    2 本の seam を共有する場合）もこの制約でそのまま正しく扱える。
  - ノード上限つき（上限到達は「未確定」扱いで、生成器は保守側 = その削りを棄却する。
    採用された盤面は必ず完走した探索で count=1）。
- **生成**: ① 全 seam をランダム彩色した完全解を作り一意性を確認（だめなら作り直し）。
  ② seam をランダム順に 1 本ずつ「なし」へ落とし、一意性が保てるときだけ採用（数独式）。
  1 パスで何も削れなくなるまで反復。時間予算内に収束すれば **局所極小**
  （どの色付き seam も単独では消せない）で、ステータスバーに表示される。
  ステージでは `maxRemove`（削り本数の上限）で難易度と決定性を制御している。
- **無推測性（パズルの手触り）**: `forcedSolvable` は「毎ステップ、置かれた隣接からの
  制約で候補 (piece, rot) がちょうど1つに定まる空きセル」を確定し続けて全マス埋まるかを
  判定する。一意解パズルでは locally-forced な確定は必ず本解の一部なので安全（順序非依存）。
  `solved=true` なら当てずっぽう不要で解ける = 良問。ステージの盤面/シードはこの指標で選定した
  （後半の広い盤はあえて `false`＝先読み要）。なお **full hints でも単一アンカー起点では
  無推測にならない**ことがあり、逆に少し間引いて空白辺を作ると強い手がかりになって
  無推測で解けるようになる、という逆説がある（Stage 9）。
- **難易度の軸**: 盤サイズだけでなく **色数 K が効く**（K が小さいほど同じ見た目のピースが増えて
  紛らわしい）。カリキュラムは色数を明示的な難易度レバーとして使い、Stage 8 で
  「同じ盤・同じ手がかりで色だけ 3→2」に落として体感させる。
- **色数の既定**: K = max(2, round(√(mn)))。情報量下限 (2K+1)² ≳ mn 由来の出発点。
